Major — should fix before merge

usePlanning.test.ts:15–19 weekDone negative-path is thin; the “count is per-week” guarantee is under-tested Effort: S
Why
The plan's test had two assertions; the written file added a good third ({ "2_0": true }, 1, 2 → 0, proving week 2's checks don't leak into week 1). But weekDone ignores itemIdx ≥ itemCount by design — there is no test that a stray key like "1_5": true beyond itemCount is not counted, which is the exact off-by-one this helper exists to prevent.
Fix
Add expect(weekDone({ "1_0": true, "1_9": true }, 1, 2)).toBe(1) — a key past the item count must be ignored.
